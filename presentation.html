Architecture

async pipeline

Parse — tree-sitter AST, zero LLM cost

Extracts functions, classes, imports, __all__ exports, and a full call graph from source. 7 languages. All graph structure is derived statically — no model involved.

Analyze — parallel, rate-aware dispatch

Files analyzed concurrently via an asyncio.Semaphore dispatcher. Results cached by SHA-256(content) — only changed files hit the LLM on re-runs.

Synthesize — grounded, not guessed

AST-extracted imports injected into every synthesis prompt as ground truth. The model cannot hallucinate dependency names — it can only use names that appear verbatim in the source.

Static analysis layer — zero LLM cost
  • →Bandit — Python security pre-pass, CWE-tagged findings
  • →Semgrep — OWASP patterns across 30+ languages
  • →Vulture — dead symbol detection at 80% confidence
  • →NetworkX — transitive dependency traversal for blast radius

Core Design Decision

Bring Your Own Key —
zero marginal cost by design

Most AI tools pay per inference call — their costs scale with every user. Arkhe is architected to avoid this entirely. Users supply their own LLM API keys. Groq and Gemini both offer free tiers, so the majority of users run Arkhe at $0 to themselves and $0 to us.

# User defines their own fallback chain ARKHE_CHAIN=openai:gpt-4o:sk-xxx,\ gemini:gemini-2.5-pro:AIza,\ groq:llama-3.3-70b:gsk # Mix any providers, any models, any order

Model Fallback Router

Priority chains per provider. Rate limit hit → model marked cooling for 10 minutes → next in chain takes over. Cooldowns persisted to SQLite across process restarts. Resilient by construction.

Four providers supported

Groq Gemini Anthropic OpenAI NVIDIA NIM

A single provider going down or rate-limiting does not interrupt an analysis.

Why this matters strategically: the BYOK model means infrastructure costs are nearly fixed regardless of how many CLI users we have. Growth doesn't create proportional cost.

Performance

Two-layer caching —
only new work gets computed

Layer 1 — Per-file SQLite cache

Every file analysis keyed by SHA-256(content). Change one file in a 200-file repo → 1 LLM call, not 200. Cache lives at <repo>/.arkhe_cache/arkhe.db and survives process restarts.

Layer 2 — Per-job result cache

Before cloning, the server queries the GitHub API for the latest commit SHA. Cache key: SHA-256(repo_url + commit_sha). Same repo at same commit → results returned instantly, zero pipeline cost. This check runs before cloning — no disk or compute cost on a hit.

Why this compounds over time
  • →Daily re-runs cost almost nothing — only the diff of changed files hits the LLM
  • →Popular repos amortize fast — the second person to analyze facebook/react at the same commit costs $0
  • →Quality scoring auto-rescores — files scoring below 0.35 are re-analyzed with an expanded token budget before synthesis
  • →Model cooldowns persisted — fallback state survives server restarts

The combined effect: a team running Arkhe daily on their active repo pays one-time analysis cost, then incremental cost per PR — not a full re-run each time.

Engineering Detail

Built to be
trustworthy, not just fast

Hallucination suppression

AST-extracted imports injected into every synthesis prompt as ground truth. The model can only use names that appear verbatim in the source — it cannot invent or guess dependency names.

Output quality scoring

Every file analysis scored 0–1 on AST name-mention ratio, length relative to file size, and structural keyword presence. Files scoring below 0.35 are automatically re-analyzed with an expanded token budget before synthesis runs.

Dead code precision

Decorator-aware: @app.route, @router.get, pytest fixtures — framework-registered functions are never flagged. Call graph propagation marks any callee live before static analysis, eliminating callback false positives.

Reasoning trace stripping

NVIDIA Nemotron-253B emits <think> blocks during generation. A regex pass (DOTALL, handles unclosed tags) strips all chain-of-thought traces before any caller sees the result — preventing leakage into output reports.

Self-tested on its own codebase. arkhe . analyzes Arkhe itself — a 60-file, multi-language codebase. All seven outputs are accurate and pass manual review. The pipeline is its own integration test.
