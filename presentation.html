The documentation paradox The teams that most need documentation — fast-moving, shipping constantly — are the ones with the least time to write it.

Parse — tree-sitter AST

Extracts functions, classes, imports, and a full call graph. 7 languages. Zero LLM cost at this stage.

Analyze — parallel LLM calls

Each file analyzed in parallel via a rate-aware dispatcher. Results cached by content hash — only changed files hit the LLM on re-runs.

Synthesize — grounded output

AST-extracted imports injected as ground truth into every synthesis prompt. The model cannot hallucinate dependency names.

Core Design Decision

Bring Your Own Key —
zero marginal cost

Most AI tools pay per inference call — their costs scale linearly with every user. Arkhe is architected to avoid this entirely.

Users supply their own LLM API keys. Groq and Gemini both offer free tiers, so the majority of users run Arkhe at $0 cost to themselves and $0 cost to us.

# User's .env — bring any combination ARKHE_CHAIN=openai:gpt-4o:sk-xxx,gemini:gemini-2.5-pro:AIza,groq:llama-3.3-70b:gsk

Model Fallback Router

Priority chains per provider. Rate limit detected → model marked cooling for 10 minutes → next model in chain takes over automatically. Cooldowns persisted to SQLite across restarts.

Multi-Provider Support

Groq Gemini Anthropic OpenAI NVIDIA NIM

Resilient by construction — a single provider going down does not interrupt an analysis.

Performance

Two-layer caching —
only new work gets computed

Layer 1 — Per-file SQLite cache

Every file analysis keyed by SHA-256(file content). Change one file in a 200-file repo → 1 LLM call, not 200. Cache lives at <repo>/.arkhe_cache/arkhe.db — survives process restarts.

Layer 2 — Per-job result cache

Before cloning, the server queries the GitHub API for the latest commit SHA. Cache key: SHA-256(repo_url + commit_sha). Same repo at same commit → results returned instantly. This check happens before cloning — zero network or disk cost on a hit.

Why this matters
  • Daily re-runs cost almost nothing — only the diff of changed files hits the LLM
  • Popular repos analyzed once — the second person to submit facebook/react at the same commit costs $0
  • Model cooldowns persisted — fallback state survives server restarts, no loss of context
  • Quality scoring auto-rescores — files scoring below 0.35 are re-analyzed once with expanded token budget, preventing low-quality synthesis

Engineering Detail

Built to produce
accurate output

Hallucination suppression

AST-extracted imports injected into every synthesis prompt as ground truth. The model cannot guess or invent dependency names — it can only use what the parser found verbatim in the source.

Quality scoring

Every file analysis scored 0–1 on AST name-mention ratio, length relative to file size, and structural keyword presence. Files scoring below 0.35 are automatically re-analyzed with an expanded token budget.

Dead code precision

Decorator-aware false positive reduction: @app.route, @router.get, pytest fixtures — framework-registered functions are never flagged. Call graph propagation marks any callee as live before static analysis runs.

Reasoning trace stripping

NVIDIA Nemotron-253B emits <think> blocks in its output. A regex pass (DOTALL, handles unclosed tags) strips all reasoning traces before any caller sees the result — preventing chain-of-thought leakage into reports.

Self-tested on its own codebase. Arkhe can analyze itself — arkhe . — and all outputs are accurate against a 60-file codebase with multiple languages and complex import structures.
