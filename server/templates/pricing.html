Common questions

What does "BYOK" mean?
Bring Your Own Key. The CLI reads your own Groq, Gemini, or Anthropic API keys from a local .env file. We never see your keys and never pay for your inference. Groq and Gemini have generous free tiers — most users pay $0.
Can I analyze private repositories?
Yes with the CLI — point it at any local directory. Private repo support on the web tier is a Pro feature (coming soon).
How does the GitLab Duo integration work?
Arkhe ships a GitLab Duo flow that runs on GitLab's compute with GitLab's AI tokens — no API keys needed. Mention the agent in any MR comment and it posts a full analysis back. See the docs →
Is my code sent to Arkhe servers?
On the web tier, your repo is cloned to an ephemeral container, analyzed, then deleted. We don't store your source code. The CLI runs entirely on your machine — nothing leaves unless you call an LLM API.
What is the /context endpoint and who is it for?
After analyzing a repo, Arkhe exposes a POST /context/{job_id} API. Give it a task description and a token budget — it returns the most relevant files ranked by relevance, truncated to fit. Built for AI coding agents (Claude Code, Copilot, Cursor) to stop wasting 70% of their context window on irrelevant files.
What is "blast radius" analysis?
The GET /impact/{job_id}?file=path endpoint traces every file that depends on the given file — directly and transitively across the full AST-parsed dependency graph. It answers "what breaks if I change this?" with a depth map, risk level, and copy-paste Markdown report. Unlike IDE "find references," this is graph traversal, not symbol search — it catches indirect dependencies too.
